# 系统架构概览

> 最后更新：2026-06-22

## 一句话定义

「吾家有老」是基于微信生态的社区养老数字化平台，采用 NestJS 后端 + 微信原生小程序 + MySQL 的三层架构。

## 为什么重要

这是项目的架构基准文档，所有技术决策和模块设计都应与本文档保持一致。

## 具体内容

### 整体架构

```
┌─────────────────┐   ┌─────────────────┐
│  老人端小程序     │   │  家属端小程序     │
│  (微信原生)       │   │  (微信原生)       │
└────────┬────────┘   └────────┬────────┘
         │                     │
         └──────────┬──────────┘
                    │ HTTPS
         ┌──────────▼──────────┐
         │  Nginx 反向代理       │
         └──────────┬──────────┘
                    │
         ┌──────────▼──────────┐
         │  NestJS 后端 API     │
         │  (Docker Container)  │
         └───┬─────────────┬───┘
             │             │
    ┌────────▼─────┐ ┌─────▼────────┐
    │  MySQL 8.0   │ │  Redis 7     │
    │  (主数据库)   │ │  (缓存/会话)  │
    └──────────────┘ └──────────────┘
```

### 模块划分

| 模块 | 职责 | 关键 API |
|------|------|----------|
| auth | 微信登录 + JWT 认证 | `/auth/elder/login`, `/auth/family/login` |
| checkin | 老人打卡 + 状态查询 | `/checkin/do`, `/checkin/today` |
| alert | 异常提醒（未打卡/紧急呼叫） | `/alerts/today-status`, `/alerts/missing` |
| service | 生活服务工单管理 | `/service/types`, `/service/order` |

### 数据加密策略

敏感字段（手机号）使用 AES-256-GCM 字段级加密，密钥由环境变量 `ENCRYPTION_KEY` 提供。

### 部署方案

使用 Docker Compose 编排所有服务（Nginx + Backend + MySQL + Redis），部署在单台云服务器上。
